代码风格规范

发表于 2019-05-30 | 分类于 工作
字数统计: 3k | 阅读时长 ≈ 11

本规范提供了一种统一的编码风格规范来编写前端代码。这使得代码具有如下的特性：

  • 其它开发者或是团队成员更容易阅读和理解。
  • IDEs 更容易理解代码，从而提供高亮、格式化等辅助功能。
  • 更容易使用现有的工具。
  • 更容易实现缓存以及代码包的分拆。

命名

文件组件命名

业务模块的命名需遵从以下原则

  • 有意义的: 当前业务模块的名词。

  • 简短: 1 到 3 个单词，最好是一个单词，如果超过一个单词必须使用”-“作为连接符。但是不建议使用单词缩写。

  • 结构：当前业务模块的显示页面放置在当前文件夹的一级，子组件放在当前文件夹下的components文件夹下。

  • 业务模块的子组件：子组件的命名应该与父组件的命名紧密耦合起来。

css 命名规范

先介绍一下目前常用的几种命名法：

  1. OOCSS （Object Oriented CSS），字面意思是面向对象的CSS，是由Nicole Sullivan提出的css理论，其主要的两个原则是：

    • Separate structure and skin（分离结构和主题）

      分离结构和主题是在于将一些视觉样式效果（例如background、color）作为单独的“主题”来应用。例如某个 class 可能存在圆角，此时不要将样式规则直接写在该 class 上，应该给当前class 再增加一个 class，然后给他写上圆角规则。例如：

    • Separate container and content（分离容器和内容）

      分离容器和内容要求使页面元素不依赖于其所处位置。其实简而言之就是不使用继承选择符或者嵌套的写法。

  2. SMACSS

    其主要原则有3条：

    • Categorizing CSS Rules（为css分类）

      SMACSS 认为 css 有 5 个类别：

      1. Base。基础样式。指的是任何场景下都会存在样式。
      2. Layout（Major Components）布局样式。指的是页面上布局相关的元素。
      3. Module（Minor Components）模块样式。这个是指页面上根据 html 结构划分出来的各个模块元素，例如：button，table 等。
      4. State 状态样式。指的就是模块样式在某种特定情况下的样式，比如说按钮被点击时的样式等。
      5. Theme 主题样式，这个很明显指的就是页面主题相关的了，比如背景色，字体色等，

    • Naming Rules（命名规则）

      这个其实很简单，其实就是要求在 css 的 class 命名时要根据Categorizing CSS Rules指定的 5 个规则来进行命名。例如：

      1. Layout：使用 l-这样的前缀，l-header l-sidebar
      2. State Rules：用is-前缀，例如：.is-active、.is-hidden。
      3. Theme Rules请使用theme-前缀，例如.theme-a-background、.theme-a-shadow。

    • Minimizing the Depth of Applicability（最小化适配深度）

      这个是要求降低html 和css的耦合度，其实就是少使用继承选择符。

  3. BEM

    BEM代表块（Block），元素（Element），修饰符（Modifier）。

    Block 是页面中独立存在的区块，可以在不同场合下被重用。每个页面都可以看做是多个Block组成。

    Element 是构成Block的元素，只有在对应Block内部才具有意义，是依赖于Block的存在。

    Modifier 是描述Block或Element的属性或状态。同一Block或Element可以有多个Modifier。

  4. NEC

    • NEC 是网易提出的一种 css 规范，包括但不限于 css 命名。在命名方面将css 划分为以下几类：

      1. 重置（reset）和默认（base）（tags）：消除默认样式和浏览器差异，并设置部分标签的初始样式，以减少后面的重复劳动！你可以根据你的网站需求设置！统一处理：建议在这个位置统一调用背景图（这里指多个布局或模块或元件共用的图）和清除浮动（这里指通用性较高的布局、模块、元件内的清除）等统一设置处理的样式！
      2. 布局（grid）（.g-）：将页面分割为几个大块，通常有头部、主体、主栏、侧栏、尾部等！
      3. 模块（module）（.m-）：通常是一个语义化的可以重复使用的较大的整体！比如导航、登录、注册、各种列表、评论、搜索等！
      4. 元件（unit）（.u-）：通常是一个不可再分的较为小巧的个体，通常被重复用于各种模块中！比如按钮、输入框、loading、图标等！
      5. 功能（function）（.f-）：为方便一些常用样式的使用，我们将这些使用率较高的样式剥离出来，按需使用，通常这些选择器具有固定样式表现，比如清除浮动等！不可滥用！
      6. 皮肤（skin）（.s-）：如果你需要把皮肤型的样式抽离出来，通常为文字色、背景色（图）、边框色等，非换肤型网站通常只提取文字色！非换肤型网站不可滥用此类！
      7. 状态（.z-）：为状态类样式加入前缀，统一标识，方便识别，她只能组合使用或作为后代出现（.u-ipt.z-dis{}，.m-list li.z-sel{}），具体详见命名规则的扩展相关项。

    • 分类的命名方法：使用单个字母+”-“为前缀

      布局（grid）（.g-）；模块（module）（.m-）；元件（unit）（.u-）；功能（function）（.f-）；皮肤（skin）（.s-）；状态（.z-）

    • 后代选择器命名

      1. 约定不以单个字母+”-“为前缀且长度大于等于2的类选择器为后代选择器，如：.item为m-list模块里的每一个项，.text为m-list模块里的文本部分：.m-list .item{}.m-list .text{}。
      2. 一个语义化的标签也可以是后代选择器，比如：.m-list li{}。
      3. 不允许单个字母的类选择器*出现。

      通过使用后代选择器的方法，你不需要考虑他的命名是否已被使用，因为他只在当前模块或元件中生效，同样的样式名可以在不同的模块或元件中重复使用，互不干扰；在多人协作或者分模块协作的时候效果尤为明显！

      后代选择器不需要完整表现结构树层级，尽量能短则短。

    • 总结一下：nec 推崇的是命名以分类（.g、.m）开头，并且后代选择器不应该继续在当前命名后继续添加，而是为其增加子类，例：

综上我们可以总结一下各种方法的共同点：

  1. 不允许使用子代选择器。
  2. 多单词命名时使用’-‘或者’_’连接，禁止使用驼峰。
  3. 推荐按照HTML 结构分类单词开头。
  4. 命名单词选择上尽量简洁易懂，并且具有代表性，但是不允许随意使用缩写。

PS：

  1. 在有重复样式时，要注意命名的通用性，因为这时的 class 命名可以重复使用，如果遇到较小差异的时候，请给存在差异的 class 再添加一个差异化的命名，并为其单独的设定样式。

  3. 如果某个元素是个全局元素，但是它恰好又在某个块内，例如在 header 里的 logo，这种情况下命名时请不要和所处的块级强关联。

优点：

  1. 在项目过大时，极大的减少了 css 命名重复的可能性。
  2. 减少了子代选择器以及后代选择器的使用，提升了css的性能。
  3. 可以获得更多的描述和更加清晰的结构，从其名字可以知道某个标记的含义。
  4. 在写样式时，自然而然的就避免了深层次嵌套的问题。
  5. 提高了模块样式的复用性。
