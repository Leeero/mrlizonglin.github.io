JS的防抖与节流

发表于 2019-11-08 | 分类于 JavaScript
字数统计: 684 | 阅读时长 ≈ 2

在平时的业务开发中，经常会有输入框搜索，或者按钮点击请求等业务场景，此时如果该事件处理函数调用的频率不加限制的话，会加重后端服务的压力，进而导致用户的体验变得很差，所以在此业务场景下，我们可以对该函数进行防抖或者节流来减少调用频率。

  1. 函数防抖（debounce）

    含义：

    ​ 防抖，指的就是在触发事件后的n秒内，该函数只会被执行一次，如果在接下来的n秒内，该函数又被触发了，那么就会再次计算函数的执行时间。简单点讲：你定义了2秒内该函数只会被执行一次，那么就从你触发的第一次开始计时2秒，如果这2s内，你再次触发了这个函数，那么就需要重新计时2s，只到在此次的2s内你不会再次触发该函数，它才会真正的去执行函数内的方法。

    常见使用场景：

    • input输入框输入搜索时。

  2. 函数节流（throttle）

    含义：

    ​ 指的是连续触发事件的时候，在n秒内只执行一次。通俗点讲就是，你定义了2s的节流时间，那么在这2s内，高频的触发函数，也只会执行第一次触发的函数事件，接下来2s内的触发事件统统都不会被执行，只到下一个2s开始，才会再次执行新开始的2s内的第一个函数事件。

    常见使用场景：

    • 用户操作保存按钮时。
    • 懒加载需要计算滚动位置时。
